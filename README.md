#  C++ Multi-threaded Socket & Stress Test Simulation

這是一個基於 C++11 與 Windows Winsock2 API 實作的主從式架構、本機壓力測試 (DDoS 模擬)實驗。

本實驗住要透過實作底層網路 Socket 通訊，探討伺服器在面對高併發連線時的資源分配與耗盡現象（如 File Descriptor 限制與連線拒絕）。

## 一、 核心功能與技術特色

* **多執行緒伺服器：** 突破單執行緒的阻塞限制，主執行緒專職 `accept` 監聽，並動態分配 `std::thread` 處理每一筆獨立連線。
* **高併發壓力測試：** 瞬間釋放數千個執行緒向伺服器發起 TCP 連線，模擬分散式阻斷服務攻擊 (DDoS) 的流量特徵。
* **原子操作戰損統計：** 運用 `<atomic>` 確保在極端多執行緒環境下的計數精準度，即時回報成功突圍與被伺服器拒絕的連線數量。
* **Windows API Integration：** 完整處理 Windows 環境下的 Winsock 初始化 (`WSAStartup`) 與資源釋放機制。

## 二、 開發環境與依賴清單

* **作業系統：** Windows 
* **編譯器：** GCC (MinGW-w64) / 支援 C++11 標準
* **依賴函式庫：** `ws2_32.lib` (Windows Sockets 2)

## 三、 快速啟動

請開啟兩個命令提示字元視窗，確保編譯器支援 `-std=c++11` 與 `-pthread`。

### 1. 編譯專案
    # 編譯伺服器端
    g++ server.cpp -o server -std=c++11 -pthread -lws2_32
    
    # 編譯壓力測試端
    g++ stress_test.cpp -o stress_test -std=c++11 -pthread -lws2_32

### 2. 啟動伺服器
在第一個終端機中啟動 Server，伺服器將會開始監聽本機的 Port 8080。
    ./server

### 3. 發動壓力測試
在第二個終端機中執行測試腳本。預設將瞬間發起 5000 個連線請求。
    ./stress_test

## 四、 觀察與學習目標

執行壓力測試後，可觀察以下系統架構現象：
1. **資源消耗：** 伺服器 CPU 使用率飆升與記憶體分配壓力。
2. **連線拒絕：** 觀察作業系統如何基於 `listen()` 的 Backlog 佇列長度限制，主動丟棄無法消化的惡意連線請求。
3. **作業系統防護機制：** 探討作業系統對單一 Process 允許開啟的最大 Socket 數量限制。

## 五、 開發動機與心得感想

### 為什麼想做這個實驗？
這個實驗的靈感來自於我在網路串流平台上看到一部影片，作者實地展示了如何利用 DDoS 攻擊癱瘓詐騙網站(他本人無公開原始碼只有提到是DDoS的攻擊方式)，後來我就想起 DDoS 是不是與我做過的 Ping 實驗雷同，且後來的實驗我也有使用SocketTest3測試UDP、TCP在內網的傳輸
。我曾想過：「如果伺服器可以選擇『不回應』，為什麼 DDoS 還是這麼可怕？」

由於身為大一的我程式還未達到這麼高深，為了解答這個疑惑，我提供我的想法讓 AI 輔助我驗證，我最想知道的就是其中原理。

### 實驗中遇到的挑戰、學習
在開發過程中，我經歷了幾次預料之外的系統崩潰與錯誤，這些都是寶貴經驗：

1. **單執行緒的致命傷：** 一開始我寫的伺服器只能同時服務一個連線，我了解到早期應用層 DoS 攻擊的原理。隨後我導入了 `std::thread` 多執行緒架構，成功讓伺服器具備高併發處理能力。
2. **作業系統的底層差異：** 實作過程中，我發現 Linux 的 `<sys/socket.h>` 無法在 Windows 上編譯。讓我研究以及加上 AI 的輔助後改寫出基於 Windows Winsock2 API 的版本，讓我對跨平台網路開發有更深的認識。
3. **Big5 編碼的「許功蓋」陷阱：** 在終端機印出戰損報告時，我遭遇了經典的中文編碼亂碼問題，讓我學到了編譯器與作業系統文字編碼的歷史淵源。
4. **系統的極限：** 當我把攻擊執行緒調高到 5000 時，我發現並不是所有的連線都能成功打進伺服器。而後加入 `<atomic>` 戰損統計器，我看見作業系統底層基於 `listen()` 佇列長度的自我保護機制。

### 總結
由於身為大一的我程式還未達到這麼高深，為了解答這個疑惑，我提供我的想法讓 AI 輔助我驗證，我最想知道的就是其中原理。
這個專案幫助我把「計算機網路」與「作業系統」的抽象概念，還有我之前的 Ping 和 SocketTest3 實驗串在一起，
